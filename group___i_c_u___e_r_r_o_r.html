Detailed Description

Driver for the ICU peripheral on RZ microprocessor. This module implements the ERROR Interface.

Overview

Error events are notified to the ICU instead of directly to the CPU. The interrupt controller unit (ICU) generate interrupt or reset signal from error event generated by CPU0/1 or various peripheral modules. Consult Section "Error Events" in the RZ microprocessor User's Manual for details.

Error event is combined as follows by ICU.

  • Error events from CR52 CPU0 are combined to CPU0_ERR0(CR520_ERR0) or CPU0_ERR1(CR520_ERR1) interrupt as error events with other error interrupts.
  • Error events from CR52 CPU1 are combined to CPU1_ERR0(CR521_ERR0) or CPU1_ERR1(CR521_ERR1) interrupt as error events with other error interrupts.
  • Error events from CA55 are combined to CA55_ERR0 or CA55_ERR1 interrupt as error events with other error interrupts.
  • Error events from peripherals are combined to PERI_ERR0 or PERI_ERR1 interrupt as error events with other error interrupts.
  • Error events from DSMIF are combined to DSMIF_ERR0 or DSMIF_ERR1 interrupt as error events with other error interrupts.
  • Error events from ENCIF are combined to ENCIF_ERR0 or ENCIF_ERR1 interrupt as error events with other error interrupts.
Note
The available error interrupts may differ between devices. A full list of supporting error interrupt signals is available in the User's Manual for your device.

Features

The ICU ERROR HAL module supports the following features:

  • Initialize the ICU to a pre-defined error configuration.
    • Configure generation an error interrupt from each error events.
    • Configure generation a reset from each error events.
  • Get or clear error event status.
  • The built-in error handler in each peripheral HAL module can be called to handle the peripheral error. User-created error handlers can also be called.
    Note
    By default setting, built-in error handler provided by each peripheral HAL module is called when peripheral errors occurs.
  • Supports configuring a user callback function, which will be invoked by the HAL module when an error interrupt is generated.
    Note
    The available error event sources may differ between devices. A full list of selectable error events is available in the User's Manual for your device or tables in Usage Notes.

Usage Notes

Status Clear

The ICU_ERROR HAL module does not clear the error status flag automatically.

To manually clear an error status flag perform the following steps:

  1. Use R_ICU_ERROR_StatusGet to get the error status currently occurring.
  2. Use R_ICU_ERROR_StatusClear to clear the desired error status flag.

Multiplex Interrupt

Since the ICU_ERROR controls the interrupt processing of the related peripheral module when an error event occurs, the PERI_ERRn interrupt calls the interrupt on each peripheral side in the error handler. Therefore, the relation between the multiplex interrupts settings on each side and the actual operation is as follows. For example, if multiplex interrupt setting (ICU_ERROR side) is enabled, multiple interrupts are always enabled for the related peripheral's interrupts, regardless of the multiplex interrupt setting (Each peripheral side). Conversely, if the multiplex interrupt setting (ICU_ERROR side) is disabled, it is depended on the multiplex interrupt setting (Each peripheral side) whether it is possible to permit multiplex interrupts.

Multiplex Interrupt Enable setting (ICU_ERROR side) Multiplex Interrupt Disable setting (ICU_ERROR side)
Multiplex Interrupt Enable setting (Each peripheral side) Multiplex interrupt operation is enabled Multiplex interrupt operation follow the settings of each peripheral side
Multiplex Interrupt Disable setting (Each peripheral side) Multiplex interrupt operation is enabled Multiplex interrupt operation is disabled

Basic Example

This is a basic example of minimal use of the R_ICU_ERROR in an application.

This example shows how to notify via a callback and clear the error flag when a CPU0_ERROR15 error occurs.

void error_callback (error_callback_args_t * p_args)
{
/* Clear the error flags. */
if ((ERROR_EVENT_CPU0 == p_args->error_event) &&
(ICU_ERROR_CPU0_ERROR_EVENT15 == p_args->error_status))
{
R_ICU_ERROR_StatusClear(&g_error_ctrl, ERROR_EVENT_CPU0, ICU_ERROR_CPU0_ERROR_EVENT15);
}
}
void simple_example ()
{
/* Open the ICU_ERROR driver to configure to occur an interrupt
* when CPU0_ERROR15(Timeout for LLPP bus) event occurs. */
fsp_err_t err = R_ICU_ERROR_Open(&g_error_ctrl, &g_error_cfg);
handle_error(err);
/* Add code here. When CPU0_ERROR15(Timeout for LLPP bus) event occurs,
* the ICU_ERROR driver notifies via a callback. */
}
